# Spec Product Artifact Map

`spec-product` writes primarily into `SPEC.md`.

## Primary artifact home

Put these in `SPEC.md`:

- product summary
- audience and trust
- primary conversion
- scope and launch success
- route groups at a product level
- content ownership and discoverability posture

## Secondary artifact impact

- `UI_PLAN.md` only needs product context when route groups or content modules
  need names.
- `CONTRACT.md` is not the home for product truth. Only note product-driven
  requirements there when they create explicit API or integration work.

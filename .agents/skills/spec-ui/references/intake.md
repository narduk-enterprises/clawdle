# Spec UI Intake

Use this intake to lock UI foundation decisions.

## Visual direction

- What should the product feel like visually?
- What references or anti-goals matter?
- Is there an existing brand or design system we must reuse?

## Theme policy

- What should `ui.colors` and semantic token usage imply?
- Is the surface light, dark, system, or mode-specific?
- What contrast or readability constraints matter?

## Layout ownership

- Which shells exist: landing, app, dashboard, admin, auth, blank?
- Which routes need `fullBleed` or page-owned framing?
- What navigation pattern is expected?

## Component and state patterns

- Which recurring component families need conventions?
- How should loading, empty, error, success, disabled, retry, and skeleton
  states look at a high level?
- Which responsive priorities matter most?

## Mockup decision

- Is the UI direction already clear enough to skip mockups?
- If not, what ambiguity should a mockup resolve?

# Spec UI Artifact Map

`spec-ui` writes primarily into `UI_PLAN.md`.

## `UI_PLAN.md`

This is the canonical home for:

- `UI Foundation Lock`
- `Visual References / Mockups`
- theme and layout ownership
- component and module conventions
- route-level UI composition context

## `SPEC.md`

Only put UI material here when it affects product scope or non-negotiable
constraints, for example "must reuse the existing dashboard shell."

## `CONTRACT.md`

UI is not the home for contract details. Only note contract impact when UI
choices force explicit uploads, search, session exchange, or similar API work.

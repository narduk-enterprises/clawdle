# Spec UX Lock Checklist

Lock these before marking the UX domain complete:

- first-visit flow
- conversion or onboarding flow
- returning-user flow
- login, callback, and protected-route behavior when relevant
- loading, empty, error, success, disabled, and retry states
- support or escalation behavior
- accessibility and reduced-motion commitments that affect behavior

On update passes:

- preserve unchanged flows explicitly
- note which routes or states changed
- call out required UI or tech follow-up when flow changes force them
